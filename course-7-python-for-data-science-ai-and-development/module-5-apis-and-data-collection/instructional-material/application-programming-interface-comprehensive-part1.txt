SECTION 1.1: WHAT IS AN API?
-----------------------------

DEFINITION
----------
An API (Application Programming Interface) lets two pieces of software talk to 
each other. An API provides a way for different software components to 
communicate with each other via inputs and outputs.

DETAILED EXPLANATION
--------------------
An API is essentially a contract or agreement between two pieces of software. 
It defines:
• What requests one piece of software can make to another
• What format those requests should be in
• What responses will be returned
• How the communication should happen

Think of an API as a waiter in a restaurant:
• You (your program) don't go into the kitchen (the internal workings of 
  another system)
• Instead, you tell the waiter (the API) what you want (inputs)
• The waiter communicates with the kitchen (the other software component)
• The waiter brings back what you requested (outputs)
• You don't need to know how the kitchen works - you just need to know how 
  to order from the menu (the API's interface)

KEY CHARACTERISTICS OF APIs:
• Communication Bridge: APIs connect different software components
• Input/Output Based: APIs work with inputs (what you send) and outputs 
  (what you receive)
• Abstraction: You don't need to know internal implementation details
• Standardized Interface: APIs provide a consistent way to interact
• Language Independent: APIs can work across different programming languages
• Reusable: Once defined, APIs can be used by multiple programs

WHAT DOES "APPLICATION PROGRAMMING INTERFACE" MEAN?
----------------------------------------------------
Let's break down the term:

1. APPLICATION
   • Refers to a software program or application
   • Could be a web application, mobile app, desktop program, etc.

2. PROGRAMMING
   • Refers to the act of writing code
   • APIs are used by programmers to build applications

3. INTERFACE
   • A point where two systems meet and interact
   • Like a user interface (UI) is where humans interact with software, 
     an API is where software interacts with other software

So an API is: "A way for programmers to make their applications interact 
with other applications or software components."

REAL-WORLD EXAMPLES OF APIs
----------------------------
APIs are everywhere in modern software development:

1. SOCIAL MEDIA APIs
   • Facebook API: Allows apps to post to Facebook, get user data
   • Twitter API: Allows apps to tweet, read tweets, get user information
   • Instagram API: Allows apps to access photos and user data

2. PAYMENT APIs
   • PayPal API: Process payments in your application
   • Stripe API: Handle credit card transactions
   • Square API: Accept payments in mobile apps

3. MAPPING APIs
   • Google Maps API: Display maps, get directions, find places
   • Mapbox API: Create custom maps and location features

4. WEATHER APIs
   • OpenWeatherMap API: Get current weather and forecasts
   • Weather.com API: Access weather data for applications

5. DATA APIs
   • CoinGecko API: Get cryptocurrency prices and market data
   • News APIs: Access news articles and headlines
   • Stock Market APIs: Get stock prices and financial data

6. CLOUD SERVICE APIs
   • AWS API: Interact with Amazon Web Services
   • Google Cloud API: Use Google Cloud services
   • Azure API: Access Microsoft Azure services

WHY ARE APIs IMPORTANT?
-----------------------
APIs are crucial because they enable:

1. MODULARITY
   • Break complex systems into smaller, manageable pieces
   • Each piece can be developed and maintained independently
   • Changes to one component don't break others (if API stays the same)

2. REUSABILITY
   • Write code once, use it many times
   • Other developers can use your API without rewriting your code
   • Build on top of existing solutions

3. EFFICIENCY
   • Don't reinvent the wheel - use existing APIs
   • Focus on your core functionality, not everything
   • Leverage specialized services (payment, maps, etc.)

4. INTEGRATION
   • Connect different systems together
   • Combine services from different providers
   • Create powerful applications by combining multiple APIs

5. SCALABILITY
   • APIs allow systems to grow and change
   • Can update internal implementation without breaking API users
   • Support multiple clients using the same API

SECTION 1.2: HOW APIs ENABLE SOFTWARE COMMUNICATION
----------------------------------------------------

DETAILED EXPLANATION
--------------------
APIs enable communication between software components through a well-defined 
set of rules and protocols. When you use an API, you have:

1. YOUR PROGRAM
   • The code you write
   • The application you're building
   • The software that needs to use another service

2. DATA
   • Information you want to process
   • Data you want to send to another system
   • Data you want to receive from another system

3. OTHER SOFTWARE COMPONENTS
   • External services or libraries
   • Other parts of your application
   • Third-party services or tools

4. THE API
   • The bridge that connects your program to other software components
   • Defines how communication happens
   • Handles the translation between your code and the other system

THE COMMUNICATION FLOW
----------------------
Here's how APIs facilitate communication:

STEP 1: YOUR PROGRAM HAS DATA
------------------------------
Your program contains data that needs to be processed or sent somewhere. 
For example:
• A list of numbers that needs to be sorted
• User information that needs to be saved
• A request for weather data

STEP 2: YOU USE THE API TO COMMUNICATE
---------------------------------------
You call the API with your data (inputs). The API acts as a messenger:
• You don't directly access the other software component
• You use the API's defined interface
• You provide the required inputs

STEP 3: THE API TRANSLATES YOUR REQUEST
----------------------------------------
The API takes your inputs and translates them into a format the other 
software component understands:
• Converts your data to the right format
• Applies any necessary transformations
• Prepares the request for the other system

STEP 4: THE OTHER SOFTWARE COMPONENT PROCESSES
-----------------------------------------------
The other software component receives the request and performs the operation:
• Processes your data
• Performs calculations or operations
• Accesses databases or services
• Executes the requested functionality

STEP 5: THE API RETURNS THE OUTPUT
-----------------------------------
The API receives the result from the other software component and returns it 
to your program:
• Converts the response to a format you can use
• Returns the output to your program
• Your program receives the processed data

VISUAL REPRESENTATION
---------------------
Here's a simple diagram of the communication flow:

    YOUR PROGRAM
         |
         | (sends inputs via API)
         |
    [API INTERFACE]
         |
         | (translates and forwards)
         |
    OTHER SOFTWARE COMPONENT
         |
         | (processes and returns results)
         |
    [API INTERFACE]
         |
         | (returns outputs)
         |
    YOUR PROGRAM (receives outputs)

EXAMPLE: CALCULATOR API
------------------------
Imagine a simple calculator API:

YOUR PROGRAM:
    numbers = [5, 10, 15, 20]
    result = calculator_api.sum(numbers)

THE API:
    • Receives: [5, 10, 15, 20]
    • Processes: Sends to calculator component
    • Calculator adds: 5 + 10 + 15 + 20 = 50
    • Returns: 50

YOUR PROGRAM RECEIVES:
    result = 50

You don't need to know:
• How the calculator component adds numbers
• What programming language it's written in
• How it stores intermediate results
• Any internal implementation details

You only need to know:
• The API function name: calculator_api.sum()
• What input it expects: a list of numbers
• What output it returns: a single number (the sum)

SECTION 1.3: THE ABSTRACTION PRINCIPLE OF APIs
-----------------------------------------------

DEFINITION
----------
Just like a function, you don't have to know how the API works internally - 
you just need to know its inputs and outputs. This is called abstraction.

DETAILED EXPLANATION
--------------------
Abstraction is one of the most important concepts in programming. It means 
hiding the complex details of how something works and only showing what you 
need to use it.

THINK OF IT LIKE DRIVING A CAR
-------------------------------
When you drive a car:
• You don't need to know how the engine works internally
• You don't need to understand the transmission system
• You don't need to know about fuel injection or spark plugs
• You just need to know:
  - Turn the key to start (input)
  - Press the gas pedal to go (input)
  - Press the brake to stop (input)
  - The car moves forward (output)

The car's "interface" (steering wheel, pedals, gear shift) is like an API - 
it hides all the complexity and gives you simple controls.

HOW ABSTRACTION WORKS IN APIs
------------------------------
When you use an API:

YOU KNOW:
• What function or method to call
• What inputs (parameters) to provide
• What outputs to expect
• How to use the returned data

YOU DON'T NEED TO KNOW:
• How the API is implemented internally
• What programming language the other component uses
• How data is stored or processed internally
• What algorithms are used
• How errors are handled internally
• Any of the complex details

BENEFITS OF ABSTRACTION
-----------------------
1. SIMPLICITY
   • Focus on what you need to do, not how it's done
   • Easier to learn and use
   • Less cognitive load

2. FLEXIBILITY
   • The internal implementation can change without affecting you
   • As long as inputs and outputs stay the same, you're fine
   • Allows for optimization and improvements behind the scenes

3. MAINTAINABILITY
   • Changes to internal code don't break your code
   • You can update your code without worrying about internals
   • Easier to debug (you know what went in and what came out)

4. COLLABORATION
   • Different teams can work on different parts
   • Frontend developers don't need to understand backend internals
   • Backend developers don't need to understand frontend internals

EXAMPLE: PANDAS ABSTRACTION
---------------------------
When you use pandas:

YOU DO:
    import pandas as pd
    data = {'name': ['Alice', 'Bob'], 'age': [25, 30]}
    df = pd.DataFrame(data)
    result = df.mean()

YOU KNOW:
• pd.DataFrame() creates a dataframe (input: dictionary)
• df.mean() calculates the mean (output: mean values)

YOU DON'T NEED TO KNOW:
• How pandas stores data internally
• What data structures it uses
• How it optimizes calculations
• What C libraries it calls
• How memory is managed
• Any of the complex internals

The pandas API abstracts all that complexity away!

SECTION 1.4: REAL-WORLD ANALOGIES FOR APIs
------------------------------------------

ANALOGY 1: RESTAURANT MENU
---------------------------
A restaurant menu is like an API:

THE MENU (API):
• Lists available dishes (available functions/methods)
• Shows prices (what inputs are needed)
• Describes dishes (what outputs you'll get)
• Doesn't show the recipe (internal implementation)

YOU (YOUR PROGRAM):
• Order from the menu (call API functions)
• Don't go into the kitchen (don't access internals)
• Receive your food (get outputs)
• Don't need to know cooking techniques (don't need internals)

THE KITCHEN (OTHER SOFTWARE COMPONENT):
• Receives your order (processes API request)
• Follows recipes (internal implementation)
• Prepares food (processes data)
• Sends food to you (returns outputs)

ANALOGY 2: REMOTE CONTROL
-------------------------
A TV remote control is like an API:

THE REMOTE (API):
• Has buttons for different functions (API methods)
• Power button (start/stop function)
• Volume buttons (increase/decrease function)
• Channel buttons (change data function)

YOU (YOUR PROGRAM):
• Press buttons (call API methods)
• Don't need to know how TV works internally
• TV responds to your commands (returns outputs)

THE TV (OTHER SOFTWARE COMPONENT):
• Receives signals from remote (processes API calls)
• Has complex internal electronics (internal implementation)
• Changes channels, volume, etc. (processes requests)
• You see the result on screen (outputs)

ANALOGY 3: BANK ATM
-------------------
An ATM machine is like an API:

THE ATM INTERFACE (API):
• Shows available operations (available methods)
• Withdraw money (one function)
• Check balance (another function)
• Deposit money (another function)

YOU (YOUR PROGRAM):
• Select an operation (call API method)
• Enter your PIN (provide inputs)
• Don't access bank's internal systems directly
• Receive money or information (get outputs)

THE BANK SYSTEM (OTHER SOFTWARE COMPONENT):
• Receives your request (processes API call)
• Verifies your account (internal processing)
• Accesses databases (internal operations)
• Returns money or balance (outputs)

ANALOGY 4: ELECTRICAL OUTLET
-----------------------------
An electrical outlet is like an API:

THE OUTLET (API):
• Standardized interface (standardized API)
• Accepts plugs (accepts inputs)
• Provides electricity (provides outputs)
• Hides all the wiring (hides internals)

YOUR DEVICE (YOUR PROGRAM):
• Plugs into outlet (connects to API)
• Doesn't need to know about electrical grid
• Receives power (gets outputs)
• Works without understanding electricity

THE POWER GRID (OTHER SOFTWARE COMPONENT):
• Complex system of generators, transformers, etc.
• Handles all the complexity internally
• Provides power through the outlet
• You don't need to understand it

KEY TAKEAWAY FROM ANALOGIES
----------------------------
All these analogies show the same pattern:
• There's an interface (API) that simplifies interaction
• You use the interface without knowing internals
• The complex system does the work behind the scenes
• You get the results you need

This is exactly how APIs work in programming!

SECTION 2.1: UNDERSTANDING API LIBRARIES
-----------------------------------------

DEFINITION
----------
An API library is a collection of pre-written code that provides an API for 
you to use. Libraries package complex functionality into easy-to-use interfaces.

DETAILED EXPLANATION
--------------------
API libraries are collections of code that:
• Provide ready-made functionality
• Expose an API for you to use
• Handle complex operations behind the scenes
• Save you from writing code from scratch

COMMON CHARACTERISTICS OF API LIBRARIES:
• Pre-written code: Someone else wrote the complex parts
• Well-documented: Usually comes with documentation
• Tested: Has been tested by many users
• Maintained: Updated and fixed by maintainers
• Reusable: Can be used in multiple projects

EXAMPLES OF API LIBRARIES
-------------------------
1. PANDAS (Python)
   • Library for data manipulation and analysis
   • Provides DataFrame API for working with data
   • Handles complex data operations internally

2. NUMPY (Python)
   • Library for numerical computing
   • Provides array API for mathematical operations
   • Optimized C code behind the scenes

3. REQUESTS (Python)
   • Library for making HTTP requests
   • Provides simple API for web communication
   • Handles complex networking internally

4. JQUERY (JavaScript)
   • Library for DOM manipulation
   • Provides simplified API for web development
   • Handles browser compatibility issues

5. REACT (JavaScript)
   • Library for building user interfaces
   • Provides component API for UI development
   • Handles virtual DOM and rendering internally

WHY USE API LIBRARIES?
----------------------
1. SAVE TIME
   • Don't reinvent the wheel
   • Use proven, tested code
   • Focus on your specific problem

2. RELIABILITY
   • Libraries are tested by many users
   • Bugs are found and fixed
   • More reliable than writing from scratch

3. PERFORMANCE
   • Often optimized for performance
   • May use faster languages (C, C++) internally
   • Better than naive implementations

4. FEATURES
   • Rich feature sets
   • Handles edge cases
   • More functionality than you'd write yourself

5. COMMUNITY
   • Large user communities
   • Help and support available
   • Regular updates and improvements

SECTION 2.2: PANDAS AS AN API EXAMPLE
--------------------------------------

DEFINITION
----------
Pandas is actually a set of software components, much of which are not even 
written in Python. You have some data. You have a set of software components. 
We use the pandas API to process the data by communicating with the other 
software components.

DETAILED EXPLANATION
--------------------
Pandas is a perfect example of an API library because:

1. IT'S A COLLECTION OF SOFTWARE COMPONENTS
   • Pandas itself is written in Python
   • But it uses many other components written in C, Cython, and other languages
   • These components handle the heavy computational work
   • Pandas provides a Python API to access these components

2. YOU HAVE DATA
   • Your data might be in a dictionary, list, CSV file, database, etc.
   • You want to process, analyze, or manipulate this data
   • Your data is separate from the processing tools

3. YOU HAVE SOFTWARE COMPONENTS
   • Pandas includes many software components for data processing
   • These components can sort, filter, aggregate, transform data
   • They're optimized for performance (often written in C)

4. YOU USE THE PANDAS API TO COMMUNICATE
   • You don't directly access the C code or internal components
   • You use pandas' Python API (DataFrame, Series, methods, etc.)
   • The API translates your Python code to the underlying components

THE ARCHITECTURE OF PANDAS
--------------------------
Here's how pandas is structured:

LAYER 1: YOUR PYTHON CODE
    You write: df.mean()

LAYER 2: PANDAS API (Python)
    Pandas receives your call and translates it

LAYER 3: PANDAS INTERNALS (Python/Cython)
    Pandas processes the request

LAYER 4: OPTIMIZED COMPONENTS (C/C++)
    Fast numerical operations happen here

LAYER 5: SYSTEM RESOURCES
    Memory, CPU, disk access

You only interact with Layer 2 (the API). Layers 3-5 are hidden from you.

WHY PANDAS USES MULTIPLE LANGUAGES
-----------------------------------
Pandas uses components written in different languages because:

1. PERFORMANCE
   • C and C++ are much faster than Python for numerical operations
   • Can process millions of rows quickly
   • Optimized algorithms run faster

2. EFFICIENCY
   • Lower-level languages use memory more efficiently
   • Better CPU utilization
   • Faster execution

3. EXISTING LIBRARIES
   • Can use existing optimized libraries (like NumPy)
   • Leverage decades of optimization work
   • Don't need to rewrite everything

4. BEST OF BOTH WORLDS
   • Python for ease of use (API layer)
   • C/C++ for performance (internal layer)
   • You get simplicity AND speed

EXAMPLE: WHAT HAPPENS WHEN YOU CALL df.mean()
----------------------------------------------
Let's trace what happens:

YOUR CODE:
    import pandas as pd
    data = {'numbers': [1, 2, 3, 4, 5]}
    df = pd.DataFrame(data)
    result = df.mean()

STEP 1: YOU CALL df.mean()
    • You're calling a method on a DataFrame object
    • This is part of pandas' API

STEP 2: PANDAS RECEIVES THE CALL
    • Pandas' Python code receives your method call
    • It understands you want the mean of the 'numbers' column

STEP 3: PANDAS TRANSLATES
    • Pandas translates your request to operations on underlying data structures
    • It accesses the actual data (stored efficiently in NumPy arrays)

STEP 4: OPTIMIZED COMPUTATION
    • The mean calculation happens in optimized C code (via NumPy)
    • This is much faster than pure Python

STEP 5: RESULT RETURNED
    • The result is calculated: (1+2+3+4+5)/5 = 3.0
    • Pandas wraps it in a Series object
    • Returns it to your code

YOU SEE:
    result = 3.0

YOU DON'T SEE:
    • The C code that calculated the mean
    • How data is stored in memory
    • The optimization techniques used
    • Any of the internal complexity

This is the power of APIs - simplicity on the outside, complexity handled 
internally!

SECTION 2.3: CREATING INSTANCES IN API TERMINOLOGY
---------------------------------------------------

DEFINITION
----------
When you create a dictionary, and then create a pandas object with the DataFrame 
constructor, in API lingo, this is an "instance." The data in the dictionary 
is passed along to the pandas API.

DETAILED EXPLANATION
--------------------
Understanding "instance" is crucial for working with APIs. Let's break this 
down step by step.

WHAT IS AN INSTANCE?
--------------------
An instance is a specific, concrete example of something. In programming:
• A class is like a blueprint (defines what something is)
• An instance is like a house built from that blueprint (a specific object)

REAL-WORLD ANALOGY
------------------
Think of a cookie cutter (class) and cookies (instances):
• The cookie cutter defines the shape (the class defines the structure)
• Each cookie made with the cutter is an instance (a specific object)
• All cookies have the same shape, but each is a separate cookie
• You can have many instances from one class

IN PANDAS TERMINOLOGY
---------------------
When you write:
    data = {'name': ['Alice', 'Bob'], 'age': [25, 30]}
    df = pd.DataFrame(data)

Here's what's happening:

STEP 1: YOU CREATE A DICTIONARY
    data = {'name': ['Alice', 'Bob'], 'age': [25, 30]}
    • This is your raw data
    • It's a Python dictionary
    • Contains the information you want to work with

STEP 2: YOU CREATE A DATAFRAME INSTANCE
    df = pd.DataFrame(data)
    • pd.DataFrame is the constructor (like a cookie cutter)
    • You call it with your data dictionary
    • It creates a specific DataFrame instance (like a cookie)
    • The variable 'df' now refers to this instance

STEP 3: THE DATA IS PASSED TO THE PANDAS API
    • Your dictionary data is passed to pandas
    • Pandas takes your data and converts it to its internal format
    • The data is now accessible through the pandas API
    • You can use pandas methods on this data

WHAT "INSTANCE" MEANS IN THIS CONTEXT
--------------------------------------
The DataFrame instance (df) is:
• A specific object created from the DataFrame class
• Contains your specific data
• Provides access to pandas API methods
• Acts as a bridge between your data and pandas functionality

MULTIPLE INSTANCES EXAMPLE
--------------------------
You can create multiple instances:

    data1 = {'numbers': [1, 2, 3]}
    df1 = pd.DataFrame(data1)  # Instance 1
    
    data2 = {'numbers': [4, 5, 6]}
    df2 = pd.DataFrame(data2)  # Instance 2
    
    # df1 and df2 are different instances
    # They contain different data
    # But they both use the same pandas API

Each instance is independent:
• df1 has its own data
• df2 has its own data
• Changes to df1 don't affect df2
• They're separate objects

HOW THE DATA FLOWS
------------------
Here's the complete flow:

YOUR DATA (Dictionary)
    ↓
    {'name': ['Alice', 'Bob'], 'age': [25, 30]}
    ↓
DATAFRAME CONSTRUCTOR (pd.DataFrame)
    ↓
    Creates instance and converts data
    ↓
DATAFRAME INSTANCE (df)
    ↓
    Contains your data in pandas format
    ↓
PANDAS API METHODS
    ↓
    You can now use: df.head(), df.mean(), etc.

THE INSTANCE AS AN INTERFACE
-----------------------------
The instance (df) is your interface to pandas:
• It holds your data
• It provides methods to work with that data
• It communicates with pandas' internal components
• It returns results to you

Think of the instance as your "remote control" for your specific data through 
the pandas API.

SECTION 2.4: COMMUNICATING WITH APIs THROUGH METHODS
----------------------------------------------------

DEFINITION
----------
When you call the method head, the dataframe communicates with the API 
displaying the first few rows of the dataframe. When you call the method mean 
the API will calculate the mean and return the values.

DETAILED EXPLANATION
--------------------
Methods are functions that belong to an object (instance). When you call a 
method on your DataFrame instance, you're communicating with the pandas API 
to perform an operation.

UNDERSTANDING METHOD CALLS
---------------------------
A method call looks like this:
    instance.method_name()

Breaking it down:
• instance: Your DataFrame object (the instance we created)
• . (dot): Accesses something belonging to the instance
• method_name(): The method you want to call
• (): Executes the method

EXAMPLE 1: THE HEAD() METHOD
-----------------------------
Let's examine what happens with df.head():

YOUR CODE:
    import pandas as pd
    data = {'name': ['Alice', 'Bob', 'Charlie', 'David', 'Eve'],
            'age': [25, 30, 35, 40, 45]}
    df = pd.DataFrame(data)
    result = df.head()

STEP-BY-STEP EXPLANATION:

STEP 1: YOU HAVE A DATAFRAME INSTANCE
    • df is your DataFrame instance
    • It contains your data: 5 rows with 'name' and 'age' columns

STEP 2: YOU CALL df.head()
    • You're calling the head() method on your DataFrame instance
    • This is a request to the pandas API

STEP 3: THE DATAFRAME COMMUNICATES WITH THE API
    • Your DataFrame instance (df) sends a message to pandas' API
    • The message says: "Show me the first few rows"
    • The API receives this request

STEP 4: THE API PROCESSES THE REQUEST
    • Pandas' internal code processes your request
    • It accesses the data stored in your DataFrame instance
    • It retrieves the first 5 rows (default for head())
    • It formats them for display

STEP 5: THE API RETURNS THE RESULT
    • The API returns a new DataFrame with the first 5 rows
    • This is assigned to 'result'
    • You can now see or use this data

WHAT YOU SEE:
    result contains:
       name  age
    0  Alice  25
    1    Bob  30
    2  Charlie 35
    3  David  40
    4    Eve  45

WHAT HAPPENED INTERNALLY (YOU DON'T SEE):
    • How pandas stored the data in memory
    • How it retrieved the rows
    • How it formatted the output
    • Any of the internal processing

EXAMPLE 2: THE MEAN() METHOD
-----------------------------
Let's examine what happens with df.mean():

YOUR CODE:
    import pandas as pd
    data = {'age': [25, 30, 35, 40, 45]}
    df = pd.DataFrame(data)
    result = df.mean()

STEP-BY-STEP EXPLANATION:

STEP 1: YOU HAVE A DATAFRAME INSTANCE
    • df is your DataFrame instance
    • It contains one column 'age' with values [25, 30, 35, 40, 45]

STEP 2: YOU CALL df.mean()
    • You're calling the mean() method on your DataFrame instance
    • This requests the pandas API to calculate the mean

STEP 3: THE DATAFRAME COMMUNICATES WITH THE API
    • Your DataFrame instance sends a message to pandas' API
    • The message says: "Calculate the mean of all numeric columns"
    • The API receives this request

STEP 4: THE API PROCESSES THE REQUEST
    • Pandas' API identifies numeric columns (age column)
    • It accesses the values: [25, 30, 35, 40, 45]
    • It sends these to optimized calculation code
    • The mean is calculated: (25+30+35+40+45)/5 = 35.0

STEP 5: THE API RETURNS THE RESULT
    • The API returns a Series object with the mean value
    • This is assigned to 'result'
    • You can now use this value

WHAT YOU SEE:
    result = age    35.0
             dtype: float64

WHAT HAPPENED INTERNALLY (YOU DON'T SEE):
    • How the calculation was optimized
    • Whether it used C code or Python
    • How memory was managed
    • Any of the internal complexity

THE COMMUNICATION PATTERN
-------------------------
This pattern repeats for every method call:

    YOUR CODE
         |
         | (calls method)
         |
    DATAFRAME INSTANCE
         |
         | (communicates with API)
         |
    PANDAS API
         |
         | (processes request)
         |
    INTERNAL COMPONENTS
         |
         | (performs operation)
         |
    PANDAS API
         |
         | (returns result)
         |
    DATAFRAME INSTANCE
         |
         | (provides result)
         |
    YOUR CODE (receives result)

MORE METHOD EXAMPLES
--------------------
Here are more examples of communicating with the pandas API:

EXAMPLE: df.info()
    • Communicates: "Show me information about this DataFrame"
    • API returns: Number of rows, columns, data types, memory usage

EXAMPLE: df.describe()
    • Communicates: "Give me statistical summary"
    • API returns: Count, mean, std, min, max, quartiles

EXAMPLE: df.sort_values('age')
    • Communicates: "Sort rows by age column"
    • API returns: New DataFrame with sorted rows

EXAMPLE: df.groupby('category').sum()
    • Communicates: "Group by category and sum values"
    • API returns: Aggregated results grouped by category

KEY TAKEAWAY
------------
Every time you call a method on your DataFrame instance, you're:
1. Sending a request to the pandas API
2. The API processes your request using internal components
3. The API returns the result to you
4. You don't need to know how it works internally

This is the power of APIs - simple method calls that hide complex operations!
